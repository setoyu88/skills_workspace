## フロントマター descriptionの書き方

### 必須要素
1. **何をするか**: スキルの主機能
2. **いつ使うか**: 適用場面
3. **トリガーフレーズ**: 「〇〇して」「〇〇したい」

### 良い例
```yaml
description: arXiv論文の検索・トレンド発見・詳細分析を行う統合スキル。論文検索、トレンド把握、論文分析、研究調査時に使用。
```

### 悪い例
```yaml
description: 論文スキル
```

### 適切な長さ
50-200文字。短すぎるとマッチしない、長すぎるとトークン浪費。
