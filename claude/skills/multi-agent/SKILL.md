# マルチエージェント協調スキル（Claude Code サブエージェント版）

Claude Code のサブエージェント機能を使い、1つの問いを複数の視点・役割に分担して並行で走らせ、結果を統合してより深い分析を得る。

> 外部CLI（Codex / Gemini）には依存しない。すべて Claude Code 内の Agent ツールで完結する。

## 使えるサブエージェント

| タイプ | 用途 |
|--------|------|
| `Explore` | 読み取り専用の広域検索。コードベース横断調査、命名規則の洗い出し |
| `Plan` | 実装方針・アーキテクチャ設計の検討。トレードオフ整理 |
| `general-purpose` | 汎用。調査・多段タスク・キーワード探索 |

## ワークフロー

### 1. リサーチモード（調査・情報収集）
1. まず自分（メインの Claude）で問いを分解し、観点を 2〜3 個に分ける
2. 観点ごとに `general-purpose` / `Explore` サブエージェントを **並行起動**（1メッセージで複数 Agent 呼び出し）
3. 各エージェントの結論を受け取り、矛盾・共通点を整理して統合

### 2. レビューモード（コード・文章のレビュー）
1. 自分でまず一次レビュー
2. 異なる観点を別エージェントに割り当てて並行起動
   - 例: 「設計・アーキテクチャ観点」と「セキュリティ・品質観点」で2体
3. 両方の指摘をマージし、重複を排除して報告

### 3. ブレインストームモード（アイデア出し・深い分析）
1. 自分でまず案を出す
2. 同じ問いを役割の異なるエージェント（例: 慎重派／挑戦派）に投げる
3. 意見の相違点・共通点を整理し、統合した結論を出す

## 実行のしかた

Agent ツールを使い、役割ごとに `subagent_type` と `prompt` を指定して **同一メッセージ内で複数同時に**起動する（依存がなければ並行が速い）。

例（概念）：
- 観点A: `subagent_type: "general-purpose"` — 「○○について技術的な観点で調査し、結論だけ返して」
- 観点B: `subagent_type: "Explore"` — 「このリポジトリで△△に関係する実装箇所を洗い出して」
- 設計判断: `subagent_type: "Plan"` — 「□□を実装する方針を、トレードオフ込みで提案して」

各エージェントの**最終メッセージだけ**が結果として返るので、要点を抽出して統合する。

## 統合のルール

- 結果は必ず統合し、ユーザーにわかりやすくまとめる
- エージェント間で結論が割れたら、両論併記し、自分の判断と根拠を添える
- 各エージェントは独立コンテキストで走る（こちらの文脈は引き継がれない）ので、prompt に必要な前提を十分に書く

## 注意

- サブエージェントの起動はコストがかかる。観点が1つで足りるなら分割せず自分で対応する
- 「とりあえず3体」ではなく、**問いを分割して意味がある時だけ**並行起動する
