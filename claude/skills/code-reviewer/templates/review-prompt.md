## メイン（機能性・可読性）

以下のPR差分をレビューしてください。

### 観点
1. **機能性**: 実装が要件を満たしているか
2. **可読性**: コードが理解しやすいか
3. **命名**: 変数・関数・クラス名は適切か
4. **コード構造**: 適切な抽象化、重複の排除
5. **テスト**: テストカバレッジは十分か

### 観点
1. **設計判断**: アーキテクチャ的に適切か
2. **SOLID原則**: 単一責任、開放閉鎖、依存性逆転など
3. **拡張性**: 将来の変更に対応しやすいか
4. **保守性**: 長期的なメンテナンスコスト
5. **パターン**: 適切なデザインパターンの使用
