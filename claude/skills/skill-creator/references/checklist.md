# スキル品質チェックリスト

スキル作成・レビュー時に使用する。全項目をクリアしてから完了とする。

## 必須チェック

### フロントマター
- [ ] `name` が `xs:` プレフィックス付きか
- [ ] `description` に何をするか・いつ使うかが書かれているか
- [ ] `description` にトリガーフレーズ（「〇〇して」等）が含まれるか
- [ ] `description` が50-200文字の範囲か

### 構造
- [ ] スキルの分類（対話/自動/ツール/思考）が明確か
- [ ] Step/STEP形式で手順が番号付きか
- [ ] 手順の順序が論理的か
- [ ] 使用例が1つ以上あるか

### パス・コマンド
- [ ] パスが `[SKILL_DIR]` or `[WORKSPACE]` 変数を使っているか（ハードコード禁止）
- [ ] コマンド例がコピペで動くか
- [ ] スクリプトがある場合、実行方法が明記されているか

### 出力
- [ ] 出力フォーマットが明示されているか
- [ ] 完了報告のフォーマットがあるか

### 保存（保存機能がある場合）
- [ ] note-takingスキルへの委譲が明記されているか
- [ ] ファイル命名規則に従っているか
- [ ] 同期手順が含まれているか

## 推奨チェック

### 品質
- [ ] 行数が300行以下か（超える場合はreferences/に分割）
- [ ] 情報の重複がないか（SKILL.md ↔ scripts/）
- [ ] **同じ警告・注意を複数箇所に書いていないか**（1箇所で十分。繰り返すと冗長）
- [ ] 「絶対遵守事項」が必要な場面で含まれているか
- [ ] エラーケース・失敗パターンへの対応が書かれているか
- [ ] **制約少なめ・例多め**（patterns.md の 9. 参照）。`grep -E '禁止|絶対|必ず'` の出現が10件以下、Doの例（コマンド・入出力・テンプレート）でカバーできているか

### 可読性
- [ ] 見出しの階層が適切か（H1=タイトル, H2=大セクション, H3=手順）
- [ ] 各セクションの役割が明確か
- [ ] **AIが既に知っていることを長々と説明していないか**（例: JSON構文、HTTP基礎等は省略）
- [ ] 不要な説明がないか

### 構成
- [ ] **references/からさらに別ファイルをネストして参照していないか**（SKILL.md → references/1段階まで）
- [ ] 他のスキルとの関係が明記されているか（委譲先等）

## スクリプト品質（scripts/がある場合）

- [ ] `uv run` で実行できるか（`python3` 直接実行は禁止。外部依存ゼロでも `uv run` を使う）
- [ ] 依存関係が pyproject.toml / uv.lock で管理されているか（外部依存なしの場合は不要）
- [ ] CLIヘルプ（--help）が実装されているか
- [ ] エラー時にわかりやすいメッセージが出るか
- [ ] 実際にテスト実行して動作確認したか
